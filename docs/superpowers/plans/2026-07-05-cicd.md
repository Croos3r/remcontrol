# CI/CD Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add GitHub Actions CI (lint/format/build on push/PR) and a tag-driven release pipeline that drafts a GitHub Release with cross-compiled server binaries (Linux, macOS x86_64, macOS aarch64, Windows MSVC) and a debug-signed Android APK.

**Architecture:** Two workflows under `.github/workflows/`. `ci.yml` runs `cargo fmt/clippy/build/test` for the server and `biome ci` + `tsc --noEmit` for the app on every push/PR. `release.yml` runs on `v*` tags: one job cross-compiles the server with `cross` (Linux + macOS) and `cargo-xwin` (Windows MSVC), one job builds the APK with Gradle, and a publish job attaches all artifacts to a draft release.

**Tech Stack:** GitHub Actions, Rust stable (edition 2024), `cross`, `cargo-xwin`, Node 20, Biome 2.5.2, Expo 57, Android Gradle Plugin + JDK 17, `softprops/action-gh-release`.

## Global Constraints

- Server is Rust, edition 2024 → toolchain must be Rust ≥ 1.85 (stable satisfies this).
- App uses Expo 57 / React Native 0.86; native module `react-native-zeroconf` requires a dev build (`expo prebuild` + Gradle), not Expo Go.
- Biome version pinned to `2.5.2`; schema URL in `app/biome.json` must use the matching `2.5.2` schema (Biome 2.x schema path: `https://biomejs.dev/schemas/2.5.2/schema.json`).
- Existing app code style: single quotes, semicolons, 2-space indent. Biome config must match so the first `biome ci` run does not reformat the whole codebase.
- Node version in CI: 20 (LTS). JDK: 17 (Android Gradle Plugin requirement).
- Server Linux binary shells out to `xdotool` at runtime — no build-time system dep. `enigo` 0.6 is pure Rust, so `cross` works without custom sysroots for all targets.
- Tag convention: `v*` (e.g. `v1.0.0`). Releases are drafts (`draft: true`), published manually.
- macOS server binaries are unsigned; release notes must tell users to run `xattr -dr com.apple.quarantine <binary>`.
- No version-sync between git tag / `app.json` / `Cargo.toml` (out of scope).
- Commit message style follows repo convention: `<type>(<scope>): <desc>` plus `Co-Authored-By: Claude <noreply@anthropic.com>` trailer.

### Task 2: Create the CI workflow

**Files:**
- Create: `.github/workflows/ci.yml`

**Interfaces:**
- Consumes: `app/biome.json` and `app/package.json` scripts (from Task 1); `server/Cargo.toml`.
- Produces: a workflow triggered on push/PR that runs server fmt/clippy/build/test and app biome/tsc.

- [ ] **Step 1: Write `.github/workflows/ci.yml`**

```yaml
name: CI

on:
  push:
    branches: ['**']
  pull_request:

concurrency:
  group: ci-${{ github.ref }}
  cancel-in-progress: true

jobs:
  server:
    name: Server (fmt, clippy, build, test)
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: server
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
        with:
          components: rustfmt, clippy
      - uses: Swatinem/rust-cache@v2
        with:
          workspaces: server
      - name: Format check
        run: cargo fmt --all -- --check
      - name: Clippy
        run: cargo clippy --all-targets -- -D warnings
      - name: Build
        run: cargo build --release
      - name: Test
        run: cargo test

  app:
    name: App (biome, tsc)
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: app
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: npm
          cache-dependency-path: app/package-lock.json
      - run: npm ci
      - name: Biome (lint + format)
        run: npx @biomejs/biome ci
      - name: Type check
        run: npx tsc --noEmit
```

### Task 3: Create the release workflow

**Files:**
- Create: `.github/workflows/release.yml`

**Interfaces:**
- Consumes: `app/biome.json` / `app/package.json` (Task 1); `server/Cargo.toml`.
- Produces: a workflow triggered on `v*` tags that builds server binaries (Linux + macOS via `cross`, Windows MSVC via `cargo-xwin`) and an Android APK (Gradle), then publishes a draft GitHub Release with all artifacts.

- [ ] **Step 1: Write `.github/workflows/release.yml`**

```yaml
name: Release

on:
  push:
    tags: ['v*']

concurrency:
  group: release-${{ github.ref }}
  cancel-in-progress: false

permissions:
  contents: write

jobs:
  server:
    name: Server binaries
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: server
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
      - uses: Swatinem/rust-cache@v2
        with:
          workspaces: server
      - name: Install cross
        run: cargo install cross
      - name: Install cargo-xwin
        run: cargo install cargo-xwin

      - name: Build Linux
        run: cross build --release --target x86_64-unknown-linux-gnu
      - name: Build macOS x86_64
        run: cross build --release --target x86_64-apple-darwin
      - name: Build macOS aarch64
        run: cross build --release --target aarch64-apple-darwin
      - name: Build Windows (MSVC via cargo-xwin)
        run: cargo xwin build --release --target x86_64-pc-windows-msvc

      - name: Stage binaries
        run: |
          mkdir -p staged
          cp ../target/x86_64-unknown-linux-gnu/release/remcontrol-server staged/remcontrol-server-x86_64-linux
          cp ../target/x86_64-apple-darwin/release/remcontrol-server staged/remcontrol-server-x86_64-macos
          cp ../target/aarch64-apple-darwin/release/remcontrol-server staged/remcontrol-server-aarch64-macos
          cp ../target/x86_64-pc-windows-msvc/release/remcontrol-server.exe staged/remcontrol-server-x86_64-windows.exe
          strip staged/remcontrol-server-x86_64-linux || true
          strip staged/remcontrol-server-x86_64-macos || true
          strip staged/remcontrol-server-aarch64-macos || true

      - uses: actions/upload-artifact@v4
        with:
          name: server-binaries
          path: server/staged/*

  app:
    name: Android APK
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: app
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: npm
          cache-dependency-path: app/package-lock.json
      - uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: '17'
      - run: npm ci
      - name: Expo prebuild (android)
        run: npx expo prebuild --platform android --no-install
      - name: Gradle assembleRelease
        run: |
          cd android
          ./gradlew assembleRelease
      - name: Stage APK
        run: |
          mkdir -p staged
          cp android/app/build/outputs/apk/release/app-release.apk staged/remcontrol-app-android.apk
          ls -la staged
      - uses: actions/upload-artifact@v4
        with:
          name: android-apk
          path: app/staged/remcontrol-app-android.apk

  release:
    name: Draft release
    needs: [server, app]
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/download-artifact@v4
        with:
          path: artifacts
      - name: List artifacts
        run: find artifacts -type f -maxdepth 3
      - uses: softprops/action-gh-release@v2
        with:
          draft: true
          generate_release_notes: true
          body: |
            ## Server binaries

            - Linux: `remcontrol-server-x86_64-linux`
            - macOS (Intel): `remcontrol-server-x86_64-macos`
            - macOS (Apple Silicon): `remcontrol-server-aarch64-macos`
            - Windows: `remcontrol-server-x86_64-windows.exe`

            ### macOS note

            The macOS binaries are unsigned. On first run you may need to clear
            the quarantine attribute:

            ```sh
            xattr -dr com.apple.quarantine remcontrol-server-*
            ```

            ### Linux note

            The Linux server needs `xdotool` installed at runtime for text
            entry (`sudo apt install xdotool` or your distro's equivalent).

            ## Android app

            `remcontrol-app-android.apk` — debug-signed, sideload by allowing
            installs from unknown sources.
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```
