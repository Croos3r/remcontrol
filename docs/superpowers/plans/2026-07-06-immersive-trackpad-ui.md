# Immersive Trackpad UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the trackpad screen (`app/src/screens/TrackpadScreen.tsx`) usable in both orientations by hiding the Android system navigation bar while the screen is active, and by auto-hiding/slimming the app's own top control bar so it stops eating usable trackpad surface.

**Architecture:** Two independent changes to one screen. (1) `expo-navigation-bar` hides the Android nav bar on mount with `overlay-swipe` behavior, restores it on unmount. (2) A pure reducer (`src/topBarVisibility.ts`, unit-testable without rendering — same pattern as `src/keys.ts`) tracks `{ visible, settingsOpen }` and is driven by drag-start, idle-timeout, reveal-tap, and settings-toggle events; `TrackpadScreen` wires it to gestures, a `setTimeout` idle timer, a reveal-tap zone, and a Reanimated fade/slide.

**Tech Stack:** Expo SDK 57, `expo-navigation-bar`, existing `react-native-gesture-handler` + `react-native-reanimated` gesture/animation stack, `vitest` for pure-logic tests.

## Global Constraints

- Expo SDK is 57 — install native deps with `npx expo install <pkg>` so versions match the SDK, not bare `npm install`.
- `android/` is gitignored and generated by `expo prebuild`; adding a new native module requires `npx expo prebuild --platform android --clean` before `npx expo run:android` picks it up (per `app/CLAUDE.md`).
- Match CI order before pushing: `npm run check` (biome), `npx tsc --noEmit`, `npm test`, `npm audit --omit=dev --audit-level=high`.
- No React Native component-testing library is set up in this repo (only pure-logic vitest, e.g. `connection.test.ts`, `keys.ts`). Follow that existing pattern: push logic that can be pure into a plain module and unit test it there; UI wiring is verified manually on-device, not with a rendering test harness.

- [ ] **Step 3: Implement the reducer**

Create `app/src/topBarVisibility.ts`:
```ts
export type TopBarState = { visible: boolean; settingsOpen: boolean };

export type TopBarAction =
  | { type: 'DRAG_START' }
  | { type: 'IDLE_TIMEOUT' }
  | { type: 'REVEAL_TAP' }
  | { type: 'SETTINGS_TOGGLE' };

export const INITIAL_TOP_BAR_STATE: TopBarState = { visible: true, settingsOpen: false };

export function reduceTopBar(state: TopBarState, action: TopBarAction): TopBarState {
  switch (action.type) {
    case 'DRAG_START':
    case 'IDLE_TIMEOUT':
      return state.settingsOpen ? state : { ...state, visible: false };
    case 'REVEAL_TAP':
      return { ...state, visible: true };
    case 'SETTINGS_TOGGLE': {
      const settingsOpen = !state.settingsOpen;
      return { visible: settingsOpen ? true : state.visible, settingsOpen };
    }
  }
}
```

> **SDK 57 API note:** The installed `expo-navigation-bar` (~57.0.1) replaced the old
> `setVisibilityAsync('hidden')` + `setBehaviorAsync('overlay-swipe')` pair with a single
> synchronous `NavigationBar.setHidden(boolean)`. The native side hardcodes
> `BEHAVIOR_SHOW_TRANSIENT_BARS_BY_SWIPE`, so swipe-to-reveal comes for free — no separate
> behavior call is needed. Import the named `NavigationBar` (not `* as`), since `setHidden`
> is a member of that namespace.
