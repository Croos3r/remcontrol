## Commands

```sh
npm install
npx expo run:android   # build + install on a plugged-in device (USB debugging on)
npm run start           # expo start
npm run check           # biome check (lint + format check)
npm run lint            # biome lint only
npm run format          # biome format --write
npx tsc --noEmit        # type check
npm test                # vitest run
npx vitest run src/keys.test.ts   # run a single test file
```

CI (`.github/workflows/ci.yml`) runs, in this order: `biome ci`, `tsc
--noEmit`, `npm test`, `npm audit --omit=dev --audit-level=high`. Match this
before pushing.

This project needs a dev build (`expo run:android`), not Expo Go: it depends
on the native module `react-native-zeroconf` for mDNS discovery.

The `android/` directory is generated by `expo prebuild` and is gitignored.
Native config changes go through `app.json` or a config plugin in `plugins/`,
never by editing `android/` directly. Changing native config (e.g.
`orientation`) requires `expo prebuild --platform android --clean` — a plain
`run:android` reuses the stale prebuilt tree.

## Architecture

- `src/connection.ts` — `Connection` class: owns the WebSocket lifecycle,
  drives the handshake via `crypto.ts`, tracks send/receive frame counters for
  replay protection, and exposes typed methods (`move`, `click`, `scroll`,
  `key`, `text`, ...) that the trackpad UI calls directly. Takes a
  `WebSocketFactory` for testability (see `connection.test.ts`).
- `src/crypto.ts` — client side of the PSK-ECDH handshake (X25519 + HKDF +
  ChaCha20-Poly1305), paired with `server/src/crypto.rs`. Any change here
  needs a matching change server-side — see the wire-protocol section in the
  repo-root CLAUDE.md.
- `src/storage.ts` — persists the last successful connection (IP/port) and
  the pairing token in Expo SecureStore (not AsyncStorage — the token is
  secret). Storage keys join host+port with `-`, not `:` — SecureStore
  rejects `:` in key names.
- `src/keys.ts` / `components/KeyPanel.tsx` — pure key/layout constants
  separated from the component so they're unit-testable without rendering.
- `screens/ConnectScreen.tsx` — QR scan, mDNS discovery list, and manual
  entry; three ways to obtain a `ServerInfo` (ip/port/token).
- `screens/TrackpadScreen.tsx` — the gesture surface (pan/tap/double-tap-hold
  for move/click/drag, 2-finger pan/tap for scroll/right-click) plus the
  keyboard toggle and docked key panel; adapts layout in landscape via
  `useIsLandscape`.

## Known platform gotchas (see also root memory notes)

- mDNS discovery requires `CHANGE_WIFI_MULTICAST_STATE` in the Android
  manifest (set via config plugin, since `android/` is generated).
- Enterprise/campus Wi-Fi (client isolation) blocks multicast entirely — mDNS
  discovery will never find anything there; QR/manual entry still work.
- Hermes has no `crypto.getRandomValues`; `react-native-get-random-values`
  must be imported before any `@noble/*` module or key generation throws.
- Android's Network Security Config `<domain>` entries don't accept CIDR
  ranges — scoping cleartext traffic by subnet silently blocks all LAN IPs
  even though a browser on the same phone can reach them fine.
